The Problem

The AI config crisis

Every AI tool has its own config format, its own location, its own conventions. Teams copy-paste across repos with no versioning, no governance, and no visibility.

🗂️
5 tools, 5 formats

Claude uses CLAUDE.md. Codex wants AGENTS.md. Cursor needs .cursor/rules/. Every tool reinvents the wheel.

📋
Copy-paste across repos

That carefully crafted system prompt lives in 12 different repos, each slightly different, all drifting.

📦
No version tracking

You can't pin, audit, or roll back AI configuration the way you would a dependency.

🔓
No trust gates

Any MCP server can run. Any hook can fire. No approval flow, no audit trail, no policy enforcement.

How It Works

Simple by design

Four steps from scattered configs to a governed, version-controlled source of truth.

1
✏️
Define

Author canonical packages — instructions, skills, hooks, MCP configs — in a single source of truth.

2
🔗
Assign

Map packages to workspaces and targets. Different teams get different provider sets from the same source.

3
⚙️
Render

Workloom renders provider-specific files — CLAUDE.md, AGENTS.md, .cursor/rules/ — deterministically.

4
🛡️
Govern

Hooks and MCP servers require approval. Drift detection alerts you when local files diverge.

Features

Everything your team needs

Built for engineering teams that take AI workflows seriously.

🔄
Provider-Aware Rendering

One canonical source renders to Claude, Codex, Copilot, Cursor, and Gemini. Change once, update everywhere.

🛡️
Trust Gates

Hooks and MCP servers require explicit approval before touching disk. Full audit trail included.

📡
Drift Detection

Know immediately when local files diverge from desired state. wl status shows you exactly what's out of sync.

🏗️
Monorepo Native

Multi-target workspaces with independent provider sets. Each team gets exactly what they need.

🔒
Privacy by Default

Telemetry never captures prompts, code, or secrets. Opt-in anonymized metrics only. You stay in control.

🏢
Enterprise Ready

Immutable versioned packages, RBAC, audit logs, and SSO integration for organizations that need it.
